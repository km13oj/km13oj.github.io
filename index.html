Process

My design process is flexible, but rigorous, always based in user research and data analysis.

The Globe and Mail News Photo Archive

Research, Design, Launch
Project Planning
  • Discuss project KPIs and align with department heads
  • Collect requirements from stakeholders
  • Internal research and fact-finding throughout the organization on previous similar projects, internal capacities
  • Create timeline and project milestones, including research design for user and industry research
  • Work with technical director to set up workflow between marketing, design team, and developers
User Research
  • Analyze broad trends and user flows relating to photo content with interal analytics tools
  • Identify stumbling blocks, triggers to conversion and increased web sessions
  • Pinpoint fallout and exit points with Omniture
  • Develop and deploy surveys to subscribers on potential archive focal points
  • Isolate target demographics and interview matching subscribers in person about their archive preferences
Internal Research and Competitive Analysis
  • Work with photo archive team to audit total archive, document trends and available content
  • Audit other large scale archives, from museums and libraries
  • Interview photo archivists and historians about their organizational methodologies
Develop Information Architecture
  • Develop four key ways of navigating the vast archive of photos
  • Map out user journeys thorugh the site
  • Coordinate with development teams in Canada and the US
  • Finalize information architecture through iterative process with developers, confirming capabilities
Journey Mapping
  • Build new user photo archive user journeys
  • Begin to wireframe elements, using existing TGAM pattenr library
  • Work with newsletter and social teams to build process back to top of funnel
  • Begin to produce documentation of functionality required for developers
User Journey Usability Testing
  • Orchestrate Facebook polls on potential collection to assess subscriber interest
  • Interview subscribers and gather feedback on initial user journeys and wireframes
  • Incorporate their feedback, begin to work towards higher fidelity interface designs with team
  • Develop soft launch plan, working with customer engagement and social teams
Wireframes
  • Develop interaction patterns and high fidelity mockups in Sketch, mobile first
  • Move to Sketch, repeat feedback process, moving toward high fidelity mockups in an iterative process
  • Develop analytics requirements, produce metrics required for each element and step in user journey
  • Collect accessibility requirements and set up quality assurance standards
Development
  • Complete technical requirements documentation
  • Deliver final design specs and patterns to development teams at four breakpoints
  • Check in with developers weekly, working toward specs
Soft Launch
  • Launch first to subscribers only on Facebook, collecting feedback in comments and via analytics
  • Refine design and work with developers to execute changes
  • Work with photo archivists and editorial to launch initial collections, based on subscriber interests
  • Work with developers to connect customer acquisition funnel to archive
  • Ensure listed bugs are annihilated
Hard Launch
  • Launch photo archive in print and online
  • Build collection schedule, coordinating with social and newsletter teams
  • Begin working on new features, including search and commenting
  • Track archive performance on internal data analytics tools, continue to tweak design
  • Report on KPIs

Research, Design, Development

Persona Building, Journey Mapping
  • Using SPICE analysis and demographic data collected, create personas of existing users
  • Work with sales and business team to develop personas of desired users
  • Map journey as it exists now
  • Run personas of desired (fictional) users and use cases through existing map to collect more data
Journey Mapping
  • Build new user experience journey, sign up to exit
  • Incorporate new rewards for steps completed
  • Redesign messaging to match new onboarding flow
  • Begin to produce documentation of functionality required for developers
Test New User Journey
  • Run user personas through proposed new user journeys
  • Compare to current user flow
  • Incorporate feedback from team
  • Compare to competitor user flows
Wireframes
  • Use paper, pencil to sketch out initial UI ideas
  • Incorporate feedback from team
  • Begin to develop style guide based on new brand identity, user research
  • Move to Sketch, repeat feedback process, moving toward high fidelity mockups in an iterative process
  • Develop analytics requirements, produce metrics required for each element and step in user journey
User Feedback
  • Publish some mockups to longtime subscribers for feedback
  • Incorporate ideas before build
Development Design
  • Work with team to choose tech stack, set up Git repo
  • Complete technical requirement documentation
  • Co-create guides for interactive elements, expected behaviour
  • Select programming languages and frameworks (React, Bootstrap, SCSS)
Prototyping
  • Translate first wireframes into HTML and SCSS (rapid prototyping)
  • Solicit feedback from other designers at incubator
  • Return to research and user personas, testing new interface
  • Refine, iterate first prototypes based on feedback until they are ready to be duplicated
  • Continue building
User Feedback
  • Send weekly screenshots of new features to users for feedback
  • Collect internal feedback in weekly design review meetings
  • Once complete sections are ready for testing, make available to all paid users
  • Track feedback on sections with FullStory, Google Analytics, surveys, and interviews (Intercom)
